# Change: Добавить поддержку поиска аналогов товаров

## Why
Текущий инструмент ищет только точные совпадения товаров с требованиями 100% совпадения технических характеристик. Это ограничивает полезность поиска, так как:
- На маркетплейсе часто нет точных аналогов требуемого товара
- Пользователю нужны и альтернативные варианты (аналоги) с похожими характеристиками
- Расширенный поиск увеличивает шансы найти подходящий товар

## What Changes
- Добавить новые колонки для сохранения аналогов (F, G, H для второго варианта и т.д.)
- Изменить алгоритм поиска: сначала ищем 1:1 совпадение, затем - аналоги (60-90% совпадение)
- Добавить флаг типа результата: "точное совпадение" vs "аналог" с указанием процента совпадения
- Сохранять несколько найденных вариантов товаров в порядке релевантности

## Impact
- Affected specs: product-matching (существующая), analog-matching (новая)
- Affected code: `mim.lua` - структура колонок, workflow, валидация
- Breaking change: новая схема сохранения результатов (поддержка нескольких вариантов)
